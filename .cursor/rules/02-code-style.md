# Code Style Rules

- Preferir componentes funcionais e simples.
- Não criar abstrações desnecessárias.
- Manter as seções como componentes pequenos (Hero, FeatureGrid, FAQ, CTA).
- Tailwind: usar classes utilitárias e evitar CSS extra.
- Nomes:
  - Componentes em PascalCase
  - Arquivos de seção em kebab-case (ex: feature-grid.tsx)
- Quando criar algo novo, sugerir também a estrutura de arquivos.
